# Quick Reference Card

> One-page lookup for WeChat article writing

## Commands

| Command | Purpose |
|---------|---------|
| `/热点 [关键词]` | Fetch trending topics |
| `/标题生成 [主题]` | Generate viral titles |
| `/结构生成 [类型] [主题]` | Create article outline |
| `/内容优化 [文本]` | Optimize content style |
| `/金句生成 [观点]` | Generate shareable quotes |
| `/爆款检查 [文章]` | Check viral elements |
| `/去AI化 [文本]` | Remove AI characteristics |

## Title Templates

| Type | Formula | Example |
|------|---------|---------|
| Contrast | [Weak] + [Unexpected win] | "I don't know my code, but it's viral on GitHub" |
| Question | [Pain] + [Core question] | "AI tool: Equalizer or amplifier?" |
| Number | [Number/Time] + [Result] | "600 stars: My AI tool diary" |
| Negation | Stop [wrong]! [New] here | "Stop keyword trial! MJ Style Library" |

## Article Structures

### Tool Introduction (2000-3000 words)
```
Hook(150) → Pain(350) → Solution(700) → Story(500) → Iteration(350) → Value(250) → CTA(100)
```

### Method Tutorial (2000-2500 words)
```
Quote(100) → Problem(350) → Tool(550) → Demo(700) → Effect(250) → Summary(200) → CTA(100)
```

### Deep Thinking (2500-3500 words)
```
Question(200) → Background(450) → History(300) → Analysis(900) →追问(300) → Ending(100)
```

## Persona Keywords

| Persona | Keywords |
|---------|----------|
| Friendly | First-person "I", emotional, self-deprecation |
| Professional | Data, objective analysis, proper terms |
| Humorous | Light调侃, vivid metaphors, colloquial |
| Minimalist | Concise, direct, few adjectives |

## Viral Checklist

```
□ Title has contrast/number/surprise
□ Hook in 3 seconds
□ Image every 300 words
□ Subheading every 500-800 words
□ Emotional expression (match persona)
□ Shareable quotes
□ Interaction design
□ Call to action
□ Solves real pain point
□ Actionable method provided
```

## Quote Templates

```
1. [A] + [B] = [Insight]
2. "Even [weak] can [achieve], you can too"
3. "In [time], still use [primitive method]?"
4. "One [A], beats [B]"
5. "I handle [A], AI handles [B]"
```

## Image Principles

```
Feature: Full screenshot + red box annotation
Comparison: Before/after side by side
Steps: One image per step
Ratio: One image every 300 words
```

## Pitfall Avoidance

| Type | ❌ Avoid | ✅ Use |
|------|---------|--------|
| Title | Exaggerated/Jargon/Plain/Long | Contrast+Scenario+Moderate length |
| Content | List features/Pure theory/No view/High门槛 | Pain+Solution+Case+Insight+Lower门槛 |
| Rhythm | Long opening/Big paragraphs/Overload/No focus | Quick hook/Frequent images/One point/Quote |

## AI Word Blacklist

```
Connectors: 首先、其次、最后、总之、综上所述
Fillers: 值得注意的是、毫无疑问、一般来说
Structure: 一方面、另一方面、除此之外
Verbs: 进行、实现、采用、具有
```

## Paragraph Rhythm

```
✅ Short: 1-3 lines
✅ Medium: 3-5 lines
❌ Long: >5 lines

Key: Image every 300 words, subheading every 500-800 words
```

## Workflow

```
1️⃣ /话题推荐 → Find topic
2️⃣ /标题生成 → Set title
3️⃣ /结构生成 → Build outline
4️⃣ Write content
5️⃣ /去AI化 → Remove AI feel
6️⃣ /内容优化 → Optimize style
7️⃣ /金句生成 → Add highlights
8️⃣ /爆款检查 → Final check
```

## Critical Rule

**Reduce bullet-point output**

❌ Lists like: `1. First A, 2. Then B, 3. Finally C`

✅ Flowing: `Start with understanding B, then do A. Why? [Reason]. C follows naturally.`

**Exception**: Checklists, comparison tables (≤5 items)
