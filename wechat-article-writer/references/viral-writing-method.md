# Viral Writing Method

> Core methodology for WeChat public account viral articles

## Title Generation

### Type 1: Contrast
```
Formula: [Weak identity] + [Unexpected achievement]
Examples:
- "I don't even know what code I wrote, but it went viral on GitHub"
- "Even I, who can't write Hello World properly, made a prompt management tool"
```

### Type 2: Question
```
Formula: [Pain point] + [Core question]
Examples:
- "Is AI tool an equalizer, or an amplifier?"
- "Designer's prompt management dilemma - how to solve?"
```

### Type 3: Number + Effect
```
Formula: [Number/Time] + [Result]
Examples:
- "600 stars behind: My AI tool development diary"
- "A weekend, I made a tool that changed my creation habit"
```

### Type 4: Negation + Reversal
```
Formula: Stop [wrong behavior]! [New method] here
Examples:
- "Stop using keywords to trial and error! MJ Style Library + NanoBanana"
```

## Article Structures

### Tool Introduction Type

```
【Hook】150 words
- Specific time/event: "This week something magical happened"
- Contrast: "I, a [identity], actually [achievement]"

【Pain Point】350 words
- Question intro: "Let me ask you a question"
- Detailed struggle description

【Solution】700 words
- Core concept (not features first)
- Comparison: Traditional vs Your method
- Three-point feature intro + screenshots

【Creation Story】Optional 500 words
- Share creation process
- AI/tool collaboration
- Mistakes made

【Iteration】350 words
- Version comparison (0.1 vs 0.4)
- User feedback improvements
- Data support

【Value升华】250 words
- Quote: "Not knowing tech is no excuse for not implementing"
- Empowerment ending

【Call to Action】100 words
- Link/project address
- "Welcome suggestions, give a ⭐️"
```

### Method Tutorial Type

```
【Quote Opening】100 words
- Reference quote: "One style reference image beats thousand keywords"

【Problem Analysis】350 words
- Reveal误区 + Examples

【Tool Introduction】550 words

【Effect Analysis】250 words

```

### Deep Thinking Type

```

- Reference opening + Create dilemma
```

## Interaction Design

### Opening Questions
```
Pain type:
- "Let me ask: How do you usually..."
- "Have you encountered..."

Guide type:
- "So the question is: A or B?"
- "Smart you will ask: Where to find..."
```

### Ending Interaction
```
Low threshold:
- "If you have this pain point, comment"
- "Any better method? Please share"

High threshold:

Call to action:

```

## Pitfalls to Avoid

### Title Pitfalls
```
❌ Too exaggerated: Doesn't match content
❌ Full jargon: Not accessible
❌ Too plain: No click desire
❌ Too long (>30 chars): Won't display fully

✅ Create contrast but stay true
✅ Has specific scenario
✅ Appropriate length
```

### Content Pitfalls
```
❌ List features: No user experience
❌ Pure theory: No case support
❌ No personal view: Echo others
❌ Raise门槛: Makes people feel "too hard"

✅ Pain + Solution
✅ Has contrast, has cases
✅ Has unique insight
✅ Lower门槛, empower readers
```

### Rhythm Pitfalls
```
❌ Long opening: 300 words before main topic
❌ Long paragraphs: Big text blocks without images
❌ Information overload: Too much in one article
❌ No focus: Reader forgets what was said

✅ Hook in 3 seconds
✅ Image every 300 words
✅ One core point per article
✅ Has shareable quotes
```
