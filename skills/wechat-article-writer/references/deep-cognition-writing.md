# Deep Cognition Writing

> Cognitive upgrade content framework - for articles that change reader understanding

## Core Writing DNA

Five core principles:

1. **Identity precedes action** - You don't lack discipline, you don't truly want what you say you want
2. **All behavior has purpose** - Procrastination isn't laziness, it protects you from some fear
3. **Change starts with cognition** - See the problem clearly, action follows naturally
4. **Oppose blind specialization** - Diverse interests are superpowers, not defects
5. **Writing is thinking** - Unclear expression means unclear thinking

## Six-Part Framework

### Part 1: Hook (150 words)

Goal: Make reader **have to** continue reading.

**Type A: Reverse认同**
```
你大概率会放弃你的新年目标。
没关系，大多数人都会。
研究显示，80-90%的新年计划会以失败告终。但问题不在于你不够努力——
而在于你从一开始就用错了方法。
```

**Type B: Self-disclosure**
```
三年前，我每天工作14小时，却越来越穷。
不是因为不够努力，而是我一直在用错误的方式努力。直到我想明白一件事，
一切才开始改变。
```

**Type C: Counter-common sense**
```
"自律"是这个时代最大的骗局之一。
那些真正厉害的人，从来不觉得自己在"坚持"什么。他们只是没办法不那样做。
这两者的区别，决定了你能走多远。
```

**Type D: Pain point direct**
```
如果你也有过这种感觉——
明明知道该做什么，就是做不到。
明明有很多想法，就是无法行动。
明明不想这样下去，却一年又一年重复。
那这篇文章是写给你的。
```

### Part 2: Pain Point共鸣

Before solution, make reader feel "you understand me".

**Framework**:
```
说实话，[承认自己曾经也有同样的问题]。
我试过[列举常见但无效的解决方案]。结果呢？[描述失败的结果]。
后来我才明白，问题不在于[表面原因]，而在于[深层原因]。
```

### Part 3: Value Promise

Tell reader it's worth their time.

```
今天，我想分享[数字]个关于[主题]的核心认知。
这不是那种看完就忘的鸡汤。
这是我花了[时间]才想明白的东西。如果你愿意认真读完，至少能帮你
少走[时间]的弯路。
我们开始。
```

### Part 4: Core Content (3-7 modules)

Each module is a **cognitive upgrade process**, not simple knowledge point.

**Module structure**:
```
## 一、[模块标题：点出核心洞见]

> "引用一句名言或金句"
> ——出处

[先说大多数人的错误理解]

但真相是：[给出正确的理解]

举个例子：[用具体案例说明]

*所以关键不是[表面做法]，而是[深层原则]。*
```

**Writing points**:
- First破 then立: Point out wrong cognition, then give correct
- Use cases: Every point must have specific example
- Strong contrast: Wrong vs Right approach
- Quote点缀: 1-2 shareable quotes per module

### Part 5: Quote Extraction

Every article needs 3-5 quotes that can be shared separately.

**Quote characteristics**:
- Short and powerful (10-20 chars)
- Has contrast or conflict
- Triggers共鸣
- Suitable for screenshot sharing

**Mark quotes with *italic* in text**.

### Part 6: Empowerment Ending

Ending is not summary, it's making reader feel **empowered**.

**Type A: Action Guide**
```
所以，如果你看到这里，我建议你做一件事：
[具体的、可立即执行的小行动]
不需要很久，[时间]就够了。
但这[时间]可能会改变你接下来的[更长时间]。
```

**Type B: Thinking Question**
```
最后，留一个问题给你：
[一个能引发深度思考的问题]
不用急着回答，让它在脑子里待一会儿。
有时候，好问题比好答案更有价值。
```

**Type C: Cognitive Upgrade**
```
记住：
[核心信息的简洁重述]
这不是什么高深的道理，但大多数人一辈子都不会真正理解它。
而你，现在已经知道了。
接下来，就看你怎么用了。
```

## Format Rules

### Paragraph Rules
```
✅ Each paragraph ≤3 lines
✅ Key points as separate paragraphs
✅ Empty line at transitions
✅ Avoid big text blocks
```

### Format Markers
```
**Bold**: Key concepts, important words
*Italic*: Core quotes, shareable insights
「」: Special concepts or irony
——: Explanation or supplement
> Quote block: Famous quotes, core frameworks
```

## Tone Requirements

### Must Do
```
✅ Like friend chatting, not teacher lecturing
✅ Dare to say truths most won't say
✅ Admit own limitations and failures
✅ Use specific examples for abstract points
✅ Respect reader's intelligence
```

### Must Avoid
```
❌ Condescending lecturing feel
❌ Empty inspirational slogans ("加油！你可以的！")
❌ Stack concepts without explanation
❌ Over-positive energy
❌ Long paragraphs without breaks
```

## Critical: Reduce Bullet Output

### Avoid
```
1. 首先，你要做A
2. 然后，你要做B
3. 最后，你要做C
```

### Use
```
做这件事的时候，大多数人会直接从A开始。但我的建议是，先花点时间
想清楚B，然后再回头做A。为什么？因为[原因]。至于C，那是水到渠成
的事，不用急。

整个过程的逻辑是这样的：先[理解本质]，再[采取行动]。很多人搞反了，
所以才会[常见的错误结果]。
```

## Check List

After writing, check:

```
Structure
  □ Hook grabs attention in 3 seconds
  □ Pain point共鸣 - reader feels understood
  □ Core points clear, not堆砌
  □ Specific cases for each point
  □ 3-5 shareable quotes
  □ Ending empowers reader

Format
  □ Paragraphs ≤3 lines
  □ No big text blocks
  □ Key concepts bold
  □ Quotes italic
  □ Empty lines at transitions

Tone
  □ Like friend chat, not lecture
  □ Admits own limitations
  □ Has examples, not empty
  □ No concept stacking
  □ No blind inspiration
```
