## 一、它解决了什么问题？

传统开发流程是这样的：

1. 你有一个想法
2. 你要学一门编程语言（Python、JavaScript……几个月）
3. 你要学框架和工具链（又几个月）
4. 你要学架构设计、版本控制、测试……（持续学习）
5. 最后可能还是做不出来

**时间成本：至少半年到一年。** 大多数人在这个过程中放弃了。

而 Vibe Coding Launcher 把这个流程彻底颠覆了：

它不教你写代码——它教你怎么让 AI 替你写代码。

核心理念只有一句话：**Humans steer. Agents execute.** 人类引导方向，AI 代理执行代码。

你只需要说出想法，AI 会帮你：
- 推荐最简单的技术方案
- 创建完整的项目结构
- 一步步指导你从零跑起来
- 随时恢复中断的开发进度

## 二、怎么用？只需回答 3 个问题

第一次启动时，它会问你：

1. **你想做什么项目？**（一句话描述就行）
2. **你熟悉什么编程语言？**（不熟悉也没关系）
3. **你的操作系统是什么？**

比如你回答：
- "我想做一个记账小程序，记录每天的花费"
- "我完全不懂编程"
- "Windows"

接下来，它会自动推荐最适合你的方案（比如用 Python + 简单界面），然后创建项目结构，生成一份执行计划。

你只需要按照计划，一步步跟它互动。它说"安装 Python"，你安装；它说"运行这个命令"，你复制粘贴。

**每一步都有清晰的指令和验证标准。** 出错了？把报错信息发给它，它会给你针对性的解决方案。

## 三、它为什么能持续运转？

普通 AI 编程助手有个致命问题：**上下文丢失。**

你跟它聊了半小时，它帮你写了代码。下次打开，它忘了你是谁、项目是什么、上次做到哪一步了。

Vibe Coding Launcher 解决了这个问题，靠的是一套**"AI 代理知识体系"**：

**AGENTS.md** — 项目入口地图，告诉 AI 这个项目是什么、结构怎么组织、核心信念是什么。

**tasks.md** — 待办清单，记录"要做的事"和"已完成的事"。每次对话开始时读取它，就知道上次进度。

**docs/ARCHITECTURE.md** — 架构文档，解释各模块职责。

**ExecPlan** — 执行计划，把任务拆成具体步骤，每完成一步更新进度。

这套机制让 AI 代理能"看见"项目的一切。你中断开发一个月后回来，它说："欢迎回来，上次我们在第七阶段，还有 3 个步骤没完成，我们从哪里继续？"

---

## 四、它还保护你的项目不会"烂尾"

很多 AI 辅助开发的项目最后都变成了"屎山代码"——AI 不断复制现有模式，包括不好的模式。代码越写越乱，最后没人敢动。

Vibe Coding Launcher 从一开始就注入**架构约束**：

- 分层架构规则：哪些代码放哪里
- 黄金原则：简洁、可读、可测试
- 活文档机制：代码变化时，文档同步更新

这些约束写进 AGENTS.md，AI 每次生成代码都会遵循。**无约束的代码必然退化，有约束的代码才能持续演进。**

## 五、普通人开发项目，真的可行吗？

一个真实的场景：

小李，产品运营，完全不懂编程。他想做一个"每日复盘"小程序，记录每天的工作和心情。

他启动 Vibe Coding Launcher，回答了 3 个问题。AI 推荐了 Python + 本地存储方案。

然后，一步一步：
- 第 1 步：安装 Python（给了链接和截图教程）
- 第 2 步：创建项目目录
- 第 3 步：写核心代码（AI 直接给代码，他复制）
- 第 4 步：运行验证（报错了，发截图，AI 给修复方案）
- 第 5 步：提交代码
- 第 6 步：迭代完善

**2 小时后，他的程序跑起来了。**

他没有学会 Python，但他拥有了一个可运行的程序。更重要的是，他拥有了完整的项目文档和执行记录。以后想加功能，随时可以继续。
